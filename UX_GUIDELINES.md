# Tribuno 2.0 - UX Guidelines

Este documento define regras de experiência de utilização para o Tribuno 2.0.

## 1. Regra Principal

Cada ecrã deve responder rapidamente a três perguntas:

1. Onde estou?
2. O que já está feito?
3. Qual é o próximo passo?

Se um ecrã não responder a estas perguntas, precisa de melhor hierarquia, melhor contexto ou menos ruído.

O Dossiê é a principal unidade de conhecimento do Tribuno. Sempre que uma página, pesquisa ou sugestão de IA estiver ligada a um tema ou problema do mandato, deve mostrar claramente o Dossiê relacionado.

## 2. Regras de Navegação

- A navegação deve privilegiar contexto em vez de menus excessivos.
- O utilizador deve conseguir voltar ao contexto anterior sem pensar.
- Breadcrumbs devem mostrar a posição dentro do mandato, assembleia, ponto ou dossiê.
- Quando uma Assembleia, ponto, documento ou compromisso alimentar um Dossiê, essa relação deve estar visível.
- A navegação principal deve representar módulos estáveis, não tarefas temporárias.
- Ações frequentes devem estar perto do objeto a que pertencem.
- Não criar atalhos globais para tudo; primeiro mostrar relações relevantes no contexto.

## 3. Regra dos 3 Cliques

Sempre que fizer sentido, uma tarefa crítica deve estar a três cliques ou menos:

- abrir uma assembleia próxima;
- consultar um ponto;
- ver documentos associados;
- abrir um rascunho;
- registar uma nota;
- encontrar um compromisso pendente.

A regra dos 3 cliques não deve criar interfaces sobrecarregadas. Se uma ação for rara ou perigosa, pode exigir mais confirmação.

## 4. Regras de Dashboard

Dashboards devem servir decisão, não decoração.

Um dashboard deve mostrar:

- estado atual;
- progresso;
- pendentes;
- alertas;
- próximo passo;
- relações importantes.
- Dossiês ativos ou críticos, quando forem relevantes para a decisão.

Evitar dashboards com métricas que não alteram decisões. Cada métrica deve responder a uma pergunta útil.

## 5. Regras para Páginas de Trabalho

Uma página de trabalho deve ter:

- header claro;
- contexto do objeto;
- Dossiê relacionado, quando existir;
- ações principais visíveis;
- informação prioritária primeiro;
- estados de preparação/conclusão;
- zona de trabalho principal;
- relações relevantes na lateral ou em secções secundárias.

Páginas de trabalho não devem parecer landing pages. Devem parecer mesas de trabalho organizadas.

## 6. Regras para IA

A IA no Tribuno:

- ajuda;
- sugere;
- resume;
- relaciona;
- explica;
- mostra fontes ou contexto usado.

A IA nunca:

- decide;
- altera dados automaticamente;
- assume posição política final;
- substitui validação do eleito;
- cria documento final sem confirmação.

Toda sugestão de IA deve deixar claro que é sugestão. O eleito deve manter o controlo.

## 7. Estados Vazios

Estados vazios devem ser curtos e úteis.

Devem explicar:

- o que ainda não existe;
- porque é útil criar;
- qual é a próxima ação, se houver.

Evitar estados vazios longos, promocionais ou repetitivos.

## 8. Erros e Mensagens

Mensagens devem ser humanas, diretas e sem alarmismo.

Boas mensagens:

- dizem o que aconteceu;
- indicam se os dados estão seguros;
- sugerem próximo passo;
- não culpam o utilizador.

Exemplo:

```text
Rascunho não encontrado.
Este rascunho pode ter sido removido ou ainda não estar disponível neste navegador.
```

## 9. Informação Antes da Procura

O Tribuno deve antecipar contexto:

- documentos relevantes no ponto;
- compromissos pendentes no dossiê;
- histórico relacionado na pesquisa;
- decisões anteriores na preparação;
- entidades ligadas ao tema.
- Dossiês relacionados como contexto principal de conhecimento.

Pesquisar deve ser uma opção, não a única forma de encontrar contexto.

## 10. Dossiês como Contexto Principal

Os Dossiês acompanham o mandato e devem funcionar como memória ativa dos temas ou problemas acompanhados pelo eleito.

Exemplos de Dossiês:

- Habitação.
- Centro de Saúde.
- Iluminação Pública.
- Orçamento 2027.

Regras:

- Assembleias alimentam Dossiês através de pontos, documentos, atas, decisões, intervenções e compromissos.
- Um Dossiê deve mostrar histórico, documentos, pessoas, entidades, compromissos e decisões relacionadas.
- Pesquisa deve destacar Dossiês quando um resultado pertence a um tema acompanhado.
- IA deve usar Dossiês como principal contexto de conhecimento antes de sugerir resumos, perguntas, riscos ou próximos passos.
- Um Dossiê não deve duplicar conteúdos; deve relacionar objetos existentes.

## 11. Princípio de Simplicidade

Cada funcionalidade nova deve reduzir esforço mental. Se acrescenta mais escolhas, mais campos ou mais passos, deve compensar com clareza real.
